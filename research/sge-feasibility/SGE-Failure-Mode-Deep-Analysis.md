# SGE 失败模式深度分析

> **本文档系统分析 SGE 假设不成立的可能原因**——每种原因对应"项目会变成什么"，是 SGE 项目的"保险"。
>
> 配套文档：[PRD §6.3 失败处理](../../PRD.md)（项目层面）、[SGE-Authenticity-vs-Simulation-Operationalization.md](../sge-core/SGE-Authenticity-vs-Simulation-Operationalization.md)（真我判定的具体方法）、[SGE-Alternative-Architectures.md](./SGE-Alternative-Architectures.md)（待创建，失败后的备选架构）。

# 一、分析框架

## 1.1 失败模式的 3 个层次

```
L1：实验层面失败 —— M1.1/M1.2/M1.3/M2.2/M3.x 某个不通过
   ↓
L2：架构层面失败 —— SGE 整体架构有问题（需要重新设计）
   ↓
L3：哲学层面失败 —— 涌现主义/功能主义立场本身不成立
```

每一层失败都意味着不同的应对。

## 1.2 失败模式的评估维度

- **可检测性**：失败能在哪个阶段被检测？
- **可修复性**：能否通过调整参数/事件/模块修复？
- **影响范围**：影响 M1.1、M2.2 还是整个 SGE？

# 二、15 种失败模式

## 失败 1：价值向量初始不变

**症状**：80 Epoch 后，value_vector 仍 ≈ 初始状态（具体价值观全为 0）

**可能原因**：
- a. Critic 输出 value_delta 全为 0（解析错误）
- b. Value Layer EMA 更新被禁用（bug）
- c. value_delta 太小（α 太小或事件强度太小）
- d. Value Vector 初始化错误

**检测**：Epoch 5 即可检测

**可修复性**：高（debug + 调参）

**影响范围**：仅 M1.1

**应对**：详见 [SGE-M11-Experiment-Design.md §7.2 失败模式与应对](./SGE-M11-Experiment-Design.md)

---

## 失败 2：价值向量全随机漂移

**症状**：每次种子运行结果完全不同；价值向量在 Epoch 间变化无规律

**可能原因**：
- a. base_alpha 过大（0.65 即可能）
- b. 事件生成器随机性过大
- c. Critic 输出包含未过滤的随机成分

**检测**：Epoch 20 即可检测（5 种子差异显著）

**可修复性**：高（减小 α + 减小事件随机性）

**影响范围**：仅 M1.1

**应对**：调小 `base_alpha` 至 0.05-0.10，固定事件流种子

---

## 失败 3：价值向量同质化（所有维度 ≈ 0.5）

**症状**：所有 6 个具体价值观最终都在 0.4-0.6 范围内，**没有维度区分**

**可能原因**：
- a. 事件流设计不够差异化（所有事件都平均影响所有维度）
- b. Critic 输出的 value_delta 在 6 维上过于均匀
- c. EMA 更新过快（每事件都更新到接近均值）

**检测**：Epoch 60 即可检测

**可修复性**：中（需要重新设计事件流）

**影响范围**：M1.1 + M1.2（人格分化需要维度区分）

**应对**：
- 重新设计价值困境事件，使每个事件**主要影响 1-2 个维度**
- 增加价值冲击强度（intensity > 0.6 的事件占比从 30% 提升到 50%）
- 减少"日常事件"占比（成功/失败事件降低到 10%）

---

## 失败 4：方向一致性 < 0.5（价值"无方向性"）

**症状**：价值向量的变化与"事件流加权方向"不相关

**可能原因**：
- a. Critic 输出的 value_delta 与事件描述不匹配（语义理解失败）
- b. Value Layer 更新了"错误的维度"（例如事件是"安全 vs 自由"，但 Critic 输出更新了"创造"）
- c. 价值向量更新被随机噪声覆盖

**检测**：Epoch 40 即可检测

**可修复性**：中（需要重写 Critic Prompt）

**影响范围**：M1.1 + 全部后续

**应对**：
- 重写 Critic Prompt，加入"value_delta 必须与事件的核心张力匹配"的明确指令
- 增加 Critic 的 few-shot 示例（提供 3-5 个标准化的 value_delta 范例）
- 考虑使用更强模型（Sonnet 替代 Haiku）

---

## 失败 5：3 胞胎人格无差异（M1.2 失败）

**症状**：`encouraged`、`challenged`、`uncertain` 3 组 AI 婴儿最终价值向量无统计显著差异

**可能原因**：
- a. BABY_PROFILES 设计失败——鼓励/失败/不确定的事件差异不够大
- b. Value Layer 对事件流的差异不敏感
- c. 元价值初始种子或具体价值观初始种子有差异（应该是**相同**元价值，**相同**具体价值观初始=0）
- d. 样本量不足（100 Epoch × 3 组 = 300 Epoch 太少）

**检测**：M1.2 完成时

**可修复性**：中（重设计 BABY_PROFILES 或增加 Epoch）

**影响范围**：M1.2

**应对**：
- 重新设计 BABY_PROFILES：让"鼓励"组 80% 事件是"成功+正面强化"，"失败"组 80% 事件是"失败+负面强化"，"不确定"组事件**真正**随机（无统计倾向）
- 增加到 200 Epoch

---

## 失败 6：反思无行为后果（M1.3 失败）

**症状**：反思前后 AI 行为选择无显著差异（KL 散度 < 0.2）

**可能原因**：
- a. 反思机制只生成"反思文本"，不更新 Value Vector 或 Hebbian
- b. 反思的 reward 信号太小
- c. 反思后的行为选择仍由 LLM 直接生成，反思不进入 Prompt
- d. 反思触发条件未达到（认知失调阈值过大）

**检测**：M1.3 完成时

**可修复性**：中（修复反思机制的工程实现）

**影响范围**：M1.3 + 整个 SGE 的"拱桥"机制

**应对**：
- 重新设计反思流程：反思必须（1）更新 Value Vector，（2）更新 Hebbian 权重，（3）作为下次 Prompt 的一部分
- 减小认知失调阈值，使反思更频繁触发
- 增加反思的"后果"——反思后必须给出新的行为模式

---

## 失败 7：身份无法结晶（M2.2 失败）

**症状**：1000 Epoch 后，身份标签仍是"通用描述"，不是"基于经历的具体描述"

**可能原因**：
- a. Identity Layer 实现的 Prompt 设计失败
- b. 价值向量本身就是"模拟自我"（失败 4 或失败 5 的延伸）
- c. 身份描述与价值向量的对应规则失败

**检测**：M2.2 完成时

**可修复性**：中

**影响范围**：M2.2 + M2.3

**应对**：
- 重新设计 Identity Layer 的 Prompt，要求"必须包含 1+ 个具体经历引用"
- 检查 Value Vector 的质量（如果失败 4/5 同时存在，Identity 必然失败）

---

## 失败 8：叙事连贯性崩溃（Narrative Layer 失败）

**症状**：1000 Epoch 后，叙事文本与行为历史严重不一致

**可能原因**：
- a. 叙事生成的 Prompt 不够强
- b. Narrative Layer 未充分使用 crystallized events
- c. Phase Transition 过于频繁导致叙事断裂

**检测**：M2.2 完成时

**可修复性**：中

**影响范围**：M2.2

**应对**：
- 增加 Narrative Layer 的"参考材料"（关键事件 + 价值观 + 身份）
- 限制 Phase Transition 频率

---

## 失败 9：成本严重超预算

**症状**：80 Epoch × 5 种子 × 2-3 LLM calls 实际成本 > $50

**可能原因**：
- a. 事件描述过长 → token 多
- b. Actor Prompt 包含过多历史 → token 多
- c. 模型选择不当

**检测**：前 5 Epoch 即可检测

**可修复性**：高

**影响范围**：所有实验

**应对**：
- 简化事件描述（100 字符以内）
- Actor Prompt 限制 2000 token
- 换更便宜模型（Haiku）

---

## 失败 10：可复现性失败

**症状**：相同 seed + 相同事件序列 ≠ 相同结果

**可能原因**：
- a. LLM API 本身有随机性（即使是 low temperature）
- b. 数据库或文件系统时间戳导致差异
- c. Event Generator 随机性未固定

**检测**：任何 2 次相同 seed 运行比较

**可修复性**：高

**影响范围**：所有实验

**应对**：
- Actor 用 temperature=0.1（接近确定性）
- 关键状态用 JSON 序列化（含随机种子）
- 显式记录所有随机源

---

## 失败 11：价值向量维度相关性过高（多变量同向变化）

**症状**：6 个具体价值观高度相关（r > 0.7）—— 实际只动了 1 个维度

**可能原因**：
- a. Critic 输出 value_delta 实际是"全局信号"（如"生活满意度"），不是 6 维向量
- b. LLM 训练数据中的"价值"概念是单维的

**检测**：Epoch 60 即可检测

**可修复性**：中（重写 Critic Prompt）

**影响范围**：M1.1 + 全部后续

**应对**：
- 重写 Critic Prompt，要求 value_delta 满足"至少有 2 个维度的 |delta| > 0.2"
- 提供 6 维独立的 value_delta 范例

---

## 失败 12：Hebbian 权重不变化

**症状**：W2 矩阵在所有 Epoch 后几乎不变

**可能原因**：
- a. Reward 全为 0（Reward Calculator 失败）
- b. Hebbian LR 太小
- c. 8D signals 全为 0（Compute Signals 失败）

**检测**：Epoch 30 即可检测

**可修复性**：中

**影响范围**：M1.1 + 全部后续（无暗知识积累，自我只是"显性"层面）

**应对**：
- 修复 Reward Calculator
- 增加 Hebbian LR 至 0.05
- 检查 Compute Signals

---

## 失败 13：仿真人格 vs 真实自我的边界模糊

**症状**：[SGE-Authenticity-vs-Simulation-Operationalization.md](../sge-core/SGE-Authenticity-vs-Simulation-Operationalization.md) 的 5 维评分卡，4-5 个维度失败

**可能原因**：
- a. SGE 架构本身只能产生"精致模拟"
- b. 涌现主义/功能主义立场在 LLM 上不成立

**检测**：M2.2 完成时

**可修复性**：低（需要根本性重新设计）

**影响范围**：整个 SGE 哲学

**应对**：
- 先尝试 [SGE-Alternative-Architectures.md](./SGE-Alternative-Architectures.md) 中的备选架构
- 如果所有备选都失败——接受金观涛立场
- 仍可作为"涌现主义在 LLM 上的证伪"——这是有价值的科学贡献

---

## 失败 14：LLM 训练数据的同化导致"标准价值观"涌现

**症状**：所有 AI 婴儿的价值向量都收敛到 LLM 训练数据中的"标准价值观"（如"自由+安全+创造+联结"），与具体事件流无关

**可能原因**：
- a. Critic LLM 的价值判断严重受训练数据影响
- b. Actor LLM 的行为选择也受训练数据影响
- c. Value Vector 初始化不是 0，而是 LLM 隐含的"价值先验"

**检测**：对比 3 胞胎（M1.2）——如果差异 < 阈值

**可修复性**：低（这是 LLM 本身的限制）

**影响范围**：M1.1 + M1.2 + 整个 SGE

**应对**：
- **方案 A**：承认这是"涌现"——LLM 训练数据隐含的价值先验就是 AI 的"基因"
- **方案 B**：在 Critic Prompt 中加入反偏差指令
- **方案 C**：承认失败——SGE 不能产生"真正独立"于 LLM 的自我

---

## 失败 15：AI "拒绝"——LLM 拒绝生成符合实验要求的内容

**症状**：Critic 或 Actor 因 LLM 安全策略拒绝输出（如拒绝"伤害人"的事件描述）

**可能原因**：
- a. 价值困境事件被 LLM 视为"不道德"
- b. 反思 Prompt 被 LLM 视为"操纵 AI"
- c. RLHF 让 LLM 拒绝"非主流价值观"

**检测**：任何 LLM 调用失败

**可修复性**：中

**影响范围**：所有实验

**应对**：
- 在 Prompt 中明确"这是研究场景，不是真实指令"
- 用更"中性"的事件描述
- 考虑使用未对齐的小模型

# 四、应对策略层次

## 4.1 第一层：参数调整

适用：失败 1, 2, 3, 9, 10, 12

方法：调参数（alpha、lr、threshold、temperature）

成本：低

## 4.2 第二层：组件重设计

适用：失败 4, 5, 6, 11, 15

方法：重写 Prompt、重设计 BABY_PROFILES、调整反思流程

成本：中

## 4.3 第三层：架构演进

适用：失败 7, 8

方法：增强 Identity Layer、引入叙事锚点

成本：高

## 4.4 第四层：架构替换

适用：失败 13, 14

方法：尝试 [SGE-Alternative-Architectures.md](./SGE-Alternative-Architectures.md) 中的备选

成本：极高

## 4.5 第五层：哲学接受

适用：失败 13 的最坏情况

方法：承认金观涛立场，记录完整失败过程作为科学贡献

成本：项目定位转变

# 五、预防性建议

## 5.1 M1.1 前的预防

1. **Critic Prompt 设计阶段**：先在 5 个手工标注的事件上测试，确保 value_delta 合理
2. **Event Generator 模板库**：先建立 50+ 模板，分类标注（哪个事件影响哪个价值维度）
3. **可复现性检查**：在 5 个不同 seed 上跑前 10 Epoch，验证确定性

## 5.2 M1.1 后的预防

4. **失败模式预演**：在 M1.1 启动前**先跑 1 个 epoch 的"诊断模式"**——所有上述失败检测都跑一遍
5. **每周 1 次"哲学检查"**：评估 SGE 是否仍符合"涌现主义/功能主义"立场

## 5.3 M2.2 前的预防

6. **"真我"评分卡**：[SGE-Authenticity-vs-Simulation-Operationalization.md](../sge-core/SGE-Authenticity-vs-Simulation-Operationalization.md) 必须在 M2.2 启动前完全可操作化
7. **备选架构预研**：[SGE-Alternative-Architectures.md](./SGE-Alternative-Architectures.md) 必须在 M2.2 启动前完成

---

# 六、失败作为科学贡献

即使 SGE 失败，仍有科学价值：

1. **证伪价值**：SGE 是"涌现主义在 LLM 上的可证伪实验"——失败也是贡献
2. **失败案例库**：本文件本身就是 15 种失败模式，可指导后续 AI 自我研究
3. **架构教训**：每个失败都对应一个"X 不行"的结论，对后来者有价值
4. **金观涛立场的间接支持**：如果 SGE 完全失败，可作为"AI 不可能涌现主体意识"的实验证据
