# 是什么（What）

## 一句话定义

把一个人（或他的 Agent）当前"会什么、怎么想、状态如何"定义成一组可测量的变量，然后设计一条路径，把他从当前状态（A）带到目标状态（B）。

## 核心概念

**认知状态向量**：用 9 个维度描述一个人在某类任务上的完整状态：

```
S_t = {K, P, M, G, A, E, W, X, U}
```

| 符号 | 含义 | 举例 |
|------|------|------|
| K | 知识 | 知道概率的加法公式 |
| P | 程序技能 | 能解二元一次方程 |
| M | 元认知/策略 | 做错时会检查步骤 |
| G | 目标/动机 | 想考过 GRE 数学 |
| A | 注意/工作记忆 | 当前认知负荷高 |
| E | 情绪/动机 | 挫败感强，想放弃 |
| W | 世界模型/信念 | 认为自己"不是学数学的料" |
| X | 外部支架 | 有 AI 助手帮忙查公式 |
| U | 不确定性 | 对 K 的判断置信度只有 60% |

**A→B 闭环**：不是一次性判断"会/不会"，而是持续循环：

```
定义目标 B → 诊断当前 A → 选择干预 → 观察变化 → 更新状态 → 验证是否达到 B
```

# 为什么（Why）

## 问题背景

当前学习系统（在线课程、AI 助教、智能导师）通常只追踪一个指标：题目正确率。但真实的学习远比这复杂——

- 一个人做错题，可能不是"不会"，而是"太累了"（A 维度）或"觉得自己学不好"（W 维度）
- 一个人用 AI 工具做对了题，不代表他自己"会了"（X 维度的代偿）
- 只补知识（K），不改变策略（M）和动机（G），学习效果不会持久

## 核心主张

> 认知状态不是单一分数，而是多层向量。学习系统的任务不是"讲内容"，而是持续估计这些状态变量，选择能最大化状态迁移概率的干预。

## 为什么需要认知架构

经典认知架构（ACT-R、Soar、LIDA、CLARION）提供了"状态变量应该有哪些层"的理论基础——记忆、目标、规则、注意、学习、元认知。它们比单纯的成绩模型更适合定义可迁移的认知状态。

## 为什么需要 Agent 视角

现代用户的认知状态不只在脑子里——笔记、工具、AI Agent 的记忆和技能都参与了认知过程。未来"一个人的认知能力"需要把人和 Agent 作为一个联合系统来评估。

# 怎么做（How）

## 框架 1：状态向量设计

每个维度必须满足三个条件：
1. **有可观测证据**——不能只靠自评，要有行为数据
2. **有估计方法**——用什么算法从证据推断状态
3. **有干预杠杆**——知道怎么改变这个维度

| 维度 | 观测证据 | 估计方法 | 干预手段 |
|------|---------|---------|---------|
| K 知识 | 解释、选择题、迁移题 | IRT、认知诊断 | 例子、反例、检索练习 |
| P 程序技能 | 解题轨迹、错误类型 | BKT/DKT | 分步练习、变式训练 |
| M 元认知 | 反思日志、错误修正行为 | 行为序列分析 | 自我解释、预测-验证 |
| G 目标 | 目标陈述、放弃点 | 偏好学习 + 问卷 | 目标重构、价值连接 |
| A 注意 | 反应时、停顿、遗漏 | 认知负荷估计 | 脚手架、分块、节奏控制 |
| E 情绪 | 自评、坚持时间、退出行为 | 情感识别 + 事件触发 | 难度调节、反馈语气 |
| W 信念 | 解释、预测、错误假设 | 信念图谱 + NLP 抽取 | 反事实案例、信念修正 |
| X 外部支架 | 工具调用、Agent 记忆 | 知识库审计 | 笔记结构、工具训练 |
| U 不确定性 | 测量误差、模型分歧 | 贝叶斯后验 + 校准曲线 | 追加诊断、人工复核 |

## 框架 2：A→B 状态转移

把学习建模为带约束的序列决策问题：

```
S_{t+1} = update(S_t, 观测_t, 干预_t, 上下文_t)
策略(S_t, B) → 选择下一步干预
```

系统目标：让目标状态 B 的后验概率随时间增加，并在新情境中保持。

## 框架 3：人-Agent 联合认知

当用户有 AI Agent 辅助时，需要区分"人会了"和"Agent 帮会了"：

| 层 | 人 | Agent | 需要回答的问题 |
|----|-----|-------|--------------|
| 记忆 | 长期记忆、工作记忆 | 向量库、笔记、日志 | 哪些知识必须内化，哪些可以外包？ |
| 技能 | 判断、解释、迁移 | 工具调用、代码、工作流 | 人需要学判断，Agent 承担执行 |
| 目标 | 价值、意图、兴趣 | 任务队列、计划、优先级 | Agent 是否真实反映人的目标？ |
| 证据 | 表现、解释、行为轨迹 | 调用日志、输出质量 | 如何区分人的掌握与 Agent 的代偿？ |

# 验证方法

## 四个核心假设

| 假设 | 测试方法 |
|------|---------|
| 多层状态比单一分数更能预测迁移 | 对比单一 BKT、多技能模型、完整状态模型的预测准确率 |
| Agent 支架必须单独建模 | 有/无 Agent 条件下比较人的独立表现 |
| LLM 不能单独做高风险学习者建模 | 比较 LLM 与 DKT 的时序一致性和校准误差 |
| B 应定义为任务族能力而非知识点 | 比较两种 B 定义对真实项目表现的预测力 |

## 建议的原型实验

选一个窄领域，定义 20-40 个任务（知识题、步骤题、解释题、迁移题、项目题），记录用户作答、解释、Agent 使用和反思文本，比较三种模型对迁移任务的预测：单一分数、多技能模型、多层认知状态模型。

---

# 风险

- **构念污染**：把"会用 Agent"误判为"人已掌握"
- **状态过拟合**：模型只预测训练题，无法泛化到真实任务
- **黑箱诊断**：LLM 给出流畅解释但缺乏校准证据
- **伦理风险**：情绪、动机属于敏感数据，需要最小化采集和用户控制
- **目标错置**：B 若由系统单方面定义，可能损害用户长期自主性

# 与 SelfLab（人工自我）的关系

| 维度 | A→B 调研 | SelfLab（SGE） |
|------|---------|---------------|
| 核心问题 | 如何迁移人的认知状态？ | AI 能否形成持续的自我？ |
| 状态含义 | 知识、技能、策略（功能性） | 人格、价值观、叙事（存在性） |
| 共享基础 | 贝叶斯状态更新、认知架构理论 | 贝叶斯状态更新、认知架构理论 |
| 可借鉴 | 带置信度的状态向量设计、相关工作矩阵 | — |
| 不可借鉴 | IRT/DKT 等具体技术 | 身份涌现、叙事形成机制 |

**一句话总结**：A→B 解决的是"人怎么学"，SGE 解决的是"AI 怎么成为自己"。两者共享认知科学的底层工具箱，但回答的是不同的问题。
