**v2 关键改进**：显式标注 3 类跨层数据流——

| 跨层数据流 | 从 | 到 | 意义 |
|----------|---|---|------|
| **1. value_delta** | Critic（感知层）| Value Layer（符号层）| Critic 的价值判断直接更新价值观向量 |
| **2. reward** | Reward Calculator（感知层）| Hebbian Learning（表达层）| 行为学习的核心驱动信号（**Hebbian-Value 双轨的关键连接**）|
| **3. frustration** | Reward Calculator（感知层）| Time Metabolism（经验层）| 挫败感驱动时间代谢更新 |

**v2 缺失但应当标注的连接**（暂未在图中显示，避免过度复杂）：
- Time Metabolism → Compute Signals（驱动神经网络的 recurrent state）
- Memory Layer → Compute Signals（提供历史 context）

**何时使用 v1 vs v2**：
- v1（无跨层流）：新人入门、了解模块归属
- v2（含跨层流）：架构审查、debug Hebbian-Value 关系

## 4 层职责一览

| 层 | 职责 | 主要模块 | 数据流向 |
|----|------|---------|---------|
| **符号层** | 高层认知抽象（"我重视什么"、"我是谁"、"我的故事"） | Value Layer、Identity Layer、Narrative Layer、Reflection Layer | 向下：被经验层填充；向下：向经验层检索 |
| **经验层** | 经历存储与检索（"发生了什么"、"什么时候"、"类似经历"） | Memory Layer、Time Metabolism、KNN Retrieval、Crystallization | 向下：被感知层填充；向上：向符号层提供素材 |
| **感知层** | 当前事件的实时解析（"现在发生什么"、"我感受如何"） | Event Generator、Critic、Thermodynamic Noise、Reward Calculator | 向下：被表达层驱动；向上：向经验层写入 |
| **表达层** | 行为输出与隐式学习（"我做什么"、"我的行为模式"） | Hebbian Learning、Actor LLM、Async Memory | 向上：驱动感知层；向下：写入长期记忆 |

## 关键设计取舍

1. **为什么分 4 层？**
   - 符号层与经验层分离：使 Identity/Narrative 这种"高层抽象"独立于"具体经历"，便于反思与重构
   - 感知层与表达层分离：实现 Critic（感知）与 Actor（表达）双 LLM 架构
   - 经验层独立：记忆是 SGE 与其他认知架构差异最小的部分（继承自 AiBeing）

2. **为什么 Reflection 在符号层底部？**
   - Reflection 是"经验 → 符号"的拱桥（参见 [SGE-Key-Insights 19](../SGE-Key-Insights.md) 发育生物学映射）
   - 它向上汇总经验，向下沉淀符号

3. **为什么 Hebbian 在表达层而非符号层？**
   - 参见 [ARCH §1.3](../ARCH.md) 核心学习机制：Hebbian 与 Value Layer 是**平行双轨**
   - Hebbian 作用于亚符号权重（W2 神经网络），属于行为模式积累
   - Value Layer 作用于显式价值观向量，属于判断能力
   - 两者状态空间不同，概念层归属不同
