## 关键发现

1. **Value magnitude 提升 50%** — 阶段 D 的 value_magnitude=0.0169 高于阶段 C 的 0.0137-0.0194。原因：Actor 的行为选择（behavior_label + intention）作为额外信号影响 Value 累积。
2. **Identity 在 100 epoch 内仍在演化** — 4 个不同 identity（每 20 epoch 触发），entropy=1.585，stability=0.333。M2.2 三胞胎实验才观测 identity 收敛。
3. **Phase Transition stub 模式不触发** — frustration 累积路径依赖 reward，stub Critic 输出的 reward 范围太小。M2.2 真实 LLM 模式 + 1000 epoch 才能观测相变。
4. **Hawking 100 epoch 不衰减** — γ=0.01/h × 100h 后 weight ≈ 0.37，仍 > 1e-4。M2.2 1000 epoch 才能观测到删除。
5. **Personality divergence = 0 是 stub 模式的预期行为** — 默认事件流下 3 个 seed 走相同路径。M2.2 三胞胎实验（encouraged/challenged/uncertain）才会有人格分化。
6. **D6 真实 LLM 放大 Value 信号 ~8 倍** — 20 epoch 真实 LLM 下 value_magnitude=0.2578（vs stub 100 epoch 的 0.0322，按 epoch 归一化仍 ~8x）。stub Critic delta 范围 [-0.05, +0.05]，真实 Critic delta 范围 [-1, +1]。**M2.2 必须用真实 LLM** 才能观测有意义的人格分化。
7. **D6 暴露并修复 3 个 stub 模式无法发现的 bug**：
   - **Identity/Narrative off-by-one** — orchestrator 用 `range(n_epochs)`（epoch 0-indexed）但触发条件 `epoch % N == 0` 在 epoch=n-1 时永不满足。改为 `(epoch + 1) % N == 0`。
   - **Orchestrator 无 per-epoch 进度** — 真实 LLM 跑 20 epoch 需 ~2 分钟，原代码无任何中间输出 → 体感"卡住"。加 `[epoch X/Y] event=... actor=... |val|=... [flags]` 行（verbose=True）。
   - **check_consistency 假设 LLM 返回 dict** — 提示词说"只输出数字"但代码期望 dict。加防御式类型分支（dict / str / number 各自分流）。

## 哲学反思

> **阶段 D 完成的真正意义**：SGE 不再是"机制集合"，而是"端到端可运行的架构"。12 步编排让 Event → Critic → Value → Memory → Signals → Actor → Hebbian → Identity → Narrative 形成完整闭环——一个 AI 婴儿可以"经历事件 → 改变价值观 → 形成身份 → 讲述叙事"。这是 M2.1 设计的核心闭环。
>
> **下一阶段（M2.2）才是真正的考验**：1000 epoch 是否能让 3 个不同事件流的 AI 婴儿形成显著不同的人格？如果能，则 SGE 的"经历 → 解释 → 人格"假设得到验证；如果不能，则需要重新审视 Value Layer / Identity Layer 的设计。
