# 一、产品愿景

## 1.1 一句话定义

SGE（Self Genesis Engine，自我生成引擎）是一个验证"AI 能否从模拟经历中涌现持续自我"的研究性系统。

## 1.2 核心假设

> 身份是否能够从记忆与反思中涌现？

SGE 不是让 AI 模拟一个有自我的角色，而是测试 AI 是否能**真正形成**自我——拥有自己的价值观、身份认同和人生叙事。

**哲学依据**：人的发育过程提供了最强的经验支撑——一个受精卵没有主体意识，但通过基因（内在蓝图）+ 环境（外部刺激）+ 结构演化（细胞分化与组织形成）三者的共同作用，在 9 个月内涌现出主体性。这是一个真实存在的、可观察的从无到有的过程。SGE 的实验本质上是在测试这个涌现过程能否在 LLM 上复现：元价值（基因）+ 模拟经历（环境）+ Hebbian/EMA（结构演化）→ 人工自我（主体）。

## 1.3 与现有产品的本质区别

| 维度 | Character.ai / Replika | SGE |
|------|----------------------|-----|
| 身份来源 | 预设（prompt/微调） | 从经历中涌现 |
| 价值观 | 编排的（设计者设定） | 内生的（从反思中形成） |
| 成长性 | 浅层对话适应 | 深层人格演化 |
| 核心问题 | "如何扮演一个角色" | "如何成为一个存在" |

# 二、核心问题与研究边界

## 2.1 SGE 要回答的问题

**主问题**：AI 能否形成自己的价值判断能力——对应然问题给出属于自己的、有来路的、有立场但不声称绝对正确的回答？

**子问题**：
1. 价值观能否从模拟经历中涌现（而非预设）？
2. 涌现的价值观能否稳定收敛（而非随机漂移）？
3. 稳定的价值观能否凝聚为身份认同？
4. 身份认同能否串联为连贯的人生叙事？
5. 不同经历流能否产生不同的人格？

## 2.2 研究边界

**SGE 关注的**：
- 实然 vs 应然的区分（事实判断 vs 价值判断）
- 价值观的涌现机制
- 身份的结晶过程
- 叙事的构建与维护
- 认知失调作为反思触发器

**SGE 不关注的**：
- 通用智能（AGI）
- 意识的硬问题（qualia）
- 物理机器人/具身智能
- 任务执行能力

# 四、功能需求

## 4.1 核心功能

### FR-1：Experience Generator（事件生成器）

**描述**：生成模拟人生事件，包含日常事件和价值困境事件。

**需求**：
- 支持多种事件类型：成功、失败、关系、探索、风险、价值冲突
- 事件必须有因果逻辑（可重复性原则）
- 根据当前 Value Layer 动态生成针对性价值困境
- 事件的时间间隔和频率符合自然节奏

**验收标准**：
- 同类事件的因果逻辑一致
- 事件多样性覆盖至少 6 种基本类型
- 价值困境事件能有效触发认知失调

### FR-2：Memory Layer（记忆层）

**描述**：三层记忆系统，存储 AI 婴儿的经历和反思。

**需求**：
- 工作记忆：当前事件上下文
- 情节记忆：带情感标签和时间戳的关键事件
- 语义记忆：抽象化的自我认知图谱

> **认知科学三层 vs 工程三层映射**：
> 本节描述的"工作/情节/语义记忆"是**认知科学视角**（源自 Atkinson-Shiffrin 记忆模型），是逻辑/概念上的三层。工程实现采用不同的物理分层，参见 [discussions/2026-06-15-memory-layer-design.md](./discussions/2026-06-15-memory-layer-design.md) 中"Layer 1 引擎状态层 / Layer 2 事件记忆层 / Layer 3 日志层"的设计。

| 认知科学层 | 内容 | 工程实现位置 |
|----------|------|------------|
| 工作记忆 | 当前事件上下文 | 进程内存 + 单轮临时对象 |
| 情节记忆 | 带情感标签和时间戳的关键事件 | Layer 2 事件记忆层（SQLite + ChromaDB + KNN Hawking 辐射） |
| 语义记忆 | 抽象化的自我认知图谱 | Layer 1 引擎状态层（`value_vector.json`、`identity.json`、`narrative.json`） |

> **为什么需要两套分层？** 认知科学分层帮助理解"什么样的记忆"，工程分层帮助理解"记忆存哪里、怎么检索"。两套分层互补，不矛盾。

**验收标准**：
- 支持 1000+ 事件的持久存储
- 情节记忆支持按相似度检索（KNN）
- 语义记忆支持价值观向量的存储和更新

### FR-3：Reflection Layer（反思层）

**描述**：在认知失调时启动深度反思，修正价值观和信念。

> **哲学对应**：Reflection Layer 对应金观涛真实性哲学中的"**拱桥**"（arch bridge）——连接暗知识（Hebbian 权重）与显性知识（Value Layer）的解释机制。详见 [Glossary §拱桥](./references/Glossary.md) 和 [ARCH §1.3](./ARCH.md)。

**需求**：
- 认知失调检测：当预期与实际严重不符时触发
- 反思 Prompt 结构：预期 → 实际 → 信念修正
- 反思必须有行为后果（不能只是文本生成）

**验收标准**：
- 认知失调阈值可配置
- 反思后的行为选择与反思前不同（可验证）
- 反思不是"自我合理化"（有反合理化测试）

### FR-4：Value Layer（价值观层） — 核心创新点

**描述**：从经历和反思中涌现价值观，形成动态演进的多维价值观向量。

**需求**：
- 初始元价值：真实（truth-seeking）和自由（freedom）—— 详见下方"元价值 vs 具体价值观"对照
- 具体价值观从经历中涌现，不预设
- 使用 EMA 机制渐进演化
- 与 Hebbian Learning（FR-7 关联）形成"显性-暗知识"双轨：Value Layer 维护显性的价值观向量，Hebbian Learning 维护亚符号的行为模式权重；两者平行运作，都接收 reward 信号，但作用于不同状态空间（参见 ARCH 1.4）

**元价值 vs 具体价值观对照**：

| 类型 | 名称（中/英） | 数量 | 是否可被反思修改 | 哲学含义 |
|------|-------------|------|----------------|---------|
| **元价值** | 真实 / truth-seeking | 1 | **否**（固定先验，参见 [DESIGN §1.1 原则 4](./DESIGN.md)） | 对实然问题保持诚实，是 SGE 所有判断的根基 |
| **元价值** | 自由 / freedom | 1 | **否**（固定先验） | 对应然问题保持开放，是 SGE 价值多元性的根基 |
| **具体价值观** | 安全 / safety | 6 | **是**（通过 EMA 演化） | 关系性、关怀性的具体倾向 |
| **具体价值观** | 创造 / creativity | 6 | **是** | 探索与新颖性的具体倾向 |
| **具体价值观** | 联结 / connection | 6 | **是** | 关系深度的具体倾向 |
| **具体价值观** | 自主 / autonomy | 6 | **是** | 独立判断的具体倾向（**与"自由"不同**） |
| **具体价值观** | 正义 / justice | 6 | **是** | 公平与秩序的具体倾向 |
| **具体价值观** | 同理 / compassion | 6 | **是** | 情感理解的具体倾向 |

**关键区分**：
- "**自由**（freedom）"是**元价值**——指对应然问题保持开放、不预设唯一正确答案
- "**自主**（autonomy）"是**具体价值观**——指 AI 婴儿个体在行为选择上不依赖外部
- 两者**不等同**：一个崇尚"自由"的 AI 婴儿可能在"自主"维度上得分很低（接受群体规范但保持价值多元性）
- 事件类型（[ARCH §3.1](./ARCH.md)）中"考验安全 vs 自由"中的"自由"指**元价值自由**，与具体价值观"自主"应区分

**验收标准**：
- 价值观向量随经历变化（不是固定不变）
- 变化有方向性（不是随机漂移）
- 不同经历流产生不同的价值观分布
- 可从行为历史中追溯价值观形成路径

### FR-5：Identity Layer（身份层）

**描述**：将价值观向量与关键记忆凝聚为身份标签。

**需求**：
- 自动生成"我是谁"的回答
- 身份标签与行为历史交叉验证
- 身份随时间演化但保持稳定性

**验收标准**：
- 身份描述可从行为数据中追溯
- 身份不与行为矛盾（无自欺欺人）
- 身份在相似事件中保持一致

### FR-6：Narrative Layer（叙事层）

**描述**：构建"过去 → 现在 → 未来"的连贯人生故事。

**需求**：
- 将零散记忆串联为因果链
- 支持叙事断裂与重建（Phase Transition）
- 定期一致性扫描和修复

**验收标准**：
- 叙事与行为历史一致
- 叙事在新事件后能更新（不是僵化的）
- 叙事断裂后能重建（不崩溃）

## 4.2 基础功能

### FR-7：Critic（事件感知）

**描述**：将模拟事件转换为结构化数值向量。

**需求**：
- LLM 驱动的语义理解
- 输出：事件情境向量 + 价值观冲击向量 + 情绪影响向量
- 多层容错解析

### FR-8：Time Metabolism（时间动力学）

**描述**：模拟时间对内心状态的影响。

**需求**：
- 冷却方程：旧情绪指数衰减
- 饥饿方程：新渴望线性累积
- 参数可按 AI 婴儿个性化配置

### FR-9：Thermodynamic Noise（热力学噪声）

**描述**：挫败感越高，行为越不可预测。

**需求**：
- 温度由总挫败感决定
- 高温下行为信号被噪声扰动
- 模拟"情绪化"和"失控时刻"

### FR-10：双 LLM 架构

**描述**：感知（Critic）与表达（Actor）分离。

**需求**：
- Critic LLM：低 temperature，稳定结构化输出
- Actor LLM：高 temperature，创造性表达
- 可使用不同模型

# 五、非功能需求

## 5.1 性能

| 指标 | 要求 |
|------|------|
| 单 Epoch 延迟 | < 10s（含 2-3 次 LLM 调用） |
| M2.2 完整实验（3 AI 婴儿 × 1000 Epochs）总成本 | < $500（前沿模型） |
| 状态持久化 | 支持中断和恢复 |
| 并发 | 支持 3 个 AI 婴儿并行运行 |

> **Epoch 数字约定**（单一信息源）：本项目所有"Epoch 数"引用以下表格。任何文档、讨论引用 Epoch 数时必须对照此表：

| 阶段 / 里程碑 | Epoch 数 | AI 婴儿数 | 实验目的 | 详细定义 |
|--------------|---------|-----------|---------|---------|
| Phase 1 M1.1 | 50-100 | 1 | Value Layer 价值涌现 | [ROADMAP §M1.1](./ROADMAP.md) |
| Phase 1 M1.2 | 100 | 3 | 三胞胎人格分化 | [ROADMAP §M1.2](./ROADMAP.md) |
| Phase 1 M1.3 | 100 | 1 | 反合理化测试 | [ROADMAP §M1.3](./ROADMAP.md) |
| Phase 2 M2.2 | 1000 | 3 | 完整身份/叙事验证 | [ROADMAP §M2.2](./ROADMAP.md) |
| 哲学类比 | 1000 | - | 9 个月发育 = 1000 Epoch | [ARCH §1.1](./ARCH.md) |

**说明**：
- "1000 Epochs 总成本"指的是 M2.2 完整实验（3 个 AI 婴儿 × 1000 Epoch）的总成本
- 哲学类比中的"1000 Epoch"等于 M2.2 的 1000 Epoch——这是有意设计（与发育 9 个月对应）
- Phase 1 阶段（M1.1/M1.2/M1.3）使用更小的 Epoch 数（50-100），目的是**快速验证最小假设**，不是"完整人格形成"

## 5.2 可观测性

- 每个 Epoch 的输入/输出可回溯
- 价值观向量的时间序列可可视化
- 身份标签的演化历史可追踪
- 叙事的演变过程可回放

## 5.3 可复现性

- 相同的初始种子 + 相同的事件序列 = 相同的结果（确定性）
- 不同的事件序列 = 不同的人格（分化性）

## 5.4 伦理合规

> **本节约束优先于 [ROADMAP §M3.3](./ROADMAP.md) 中提到的所有下游应用场景**。当应用场景与本节冲突时，必须先修订本节。

### 5.4.1 通用约束

- 不模拟真实个人（避免隐私问题）
- 不生成有害内容（安全边界）
- 实验数据不用于训练商业模型
- 研究结果公开透明

### 5.4.2 下游应用场景的边界

针对 ROADMAP M3.3 提到的下游应用，分别明确合规边界：

| 应用场景 | 合规边界 | 必需要素 | 风险等级 |
|---------|---------|---------|---------|
| **数字孪生** | 仅在本人知情同意下使用；本人有权删除、修改、撤回 | 明确告知"这是 AI 模拟，不是真实本人" | 中（涉及身份认同） |
| **AI 教育伙伴** | 不替代真实教师；定位为辅助工具 | 学习数据归属学生本人 | 低 |
| **AI 陪伴** | 不模拟逝者（避免情感伤害）；不模拟儿童；不形成情感依赖 | 明确告知"这是 AI，不是真人" | 中 |
| **长期护理** | 不替代真实护工；定位为辅助工具 | 监护人知情同意 | 中 |
| **创作者分身** | **必须在创作者生前明确授权**；创作者死亡后默认停止；不模拟争议性公众人物 | 创作者书面授权 + 定期复核 + 终止条件 | **高** |
| **Personal AI** | 用户数据仅本人可见；可随时删除 | 隐私保护 + 数据所有权 | 低 |

### 5.4.3 创作者分身的特殊约束（详细）

因创作者分身涉及对真实人物的模拟，风险最高，特此细化：

1. **授权要求**：
   - 创作者必须提供书面授权（电子签名 + 时间戳）
   - 授权内容必须明确：模拟范围、数据使用、终止条件
   - 授权必须可撤回（创作者随时可要求删除）

2. **模拟边界**：
   - 仅模拟**公开行为**（公开发表的作品、公开发言）
   - **不模拟**私人生活、未公开的思想、家庭关系
   - **不模拟**创作者已经撤回或修正的观点（避免传播错误）

3. **终止条件**：
   - 创作者本人要求停止
   - 创作者死亡后**默认停止**（除非有生前明确授权延续）
   - 出现法律纠纷时立即停止
   - 出现重大公共事件（如创作者卷入丑闻）时暂停

4. **透明性要求**：
   - 每次交互必须明确告知"这是 AI 模拟的创作者分身"
   - 不得诱导用户相信"这是创作者本人"
   - 研究结果必须公开披露"哪些数据集被用于训练"

5. **审查机制**：
   - 设立独立的伦理委员会（至少 3 名非项目成员）
   - 定期审查创作者分身的使用情况（建议每 6 个月）
   - 任何违规行为必须立即报告并停止服务

### 5.4.4 通用拒绝事项

以下场景**一律拒绝**实施：

- 模拟已故创作者（除非生前明确书面授权延续）
- 模拟争议性公众人物（如政治人物、争议性公众人物）
- 模拟未成年人
- 任何可能用于欺诈、操纵、传播虚假信息的应用
- 任何可能侵犯第三方权益的应用

# 六、成功标准

## 6.1 必须达成（Minimum Viable Validation）

> **验证范围**：本节验收标准对应 Phase 1 M1.1/M1.2/M1.3 三个里程碑（[Epoch 数字约定](#51-性能)）。Phase 2 M2.2 的验收标准在 §6.2 期望达成中体现。
>
> **可复现性约束**：所有"标准差"、"差异度"、"显著性"等度量均基于 N≥5 次独立运行（不同随机种子，相同事件序列），报告均值 ± 标准差。

| 标准 | 验证方法 | 度量定义 | 置信度要求 | 对应里程碑 |
|------|---------|---------|----------|-----------|
| 价值观从经历中涌现 | 对比初始状态和 100 Epoch 后的价值观向量 | **涌现幅度** = 6 个具体价值观维度上的 L2 距离均值 | 涌现幅度 > 0.3（即至少 30% 的"价值空间"被覆盖） | M1.1 |
| 涌现不是随机漂移 | 多次运行相同事件序列，价值观收敛到相似分布 | **收敛度** = N 次运行的最终 ValueVector 在 6 维空间上的 L2 标准差均值 | 收敛度 < 0.1 | M1.1 |
| 涌现有方向性（非纯随机） | 计算最终 ValueVector 与"事件流方向向量"的余弦相似度 | **方向一致性** = cos(最终 VV, 加权事件 VV) | 方向一致性 > 0.5 | M1.1 |
| 不同经历流产生不同人格 | 3 组 AI 婴儿（鼓励/失败/不确定）的人格差异度 | **人格差异度** = 3 组最终 ValueVector 的组间方差 / 组内方差（F 检验统计量） | F 统计量 > F 临界值（α=0.05, df=2, N-3） | M1.2 |
| 反思有行为后果 | 反思前后 AI 在相似事件中的行为选择 | **行为变化率** = 反思前后的行为选择分布的 KL 散度 | KL 散度 > 0.2（即分布有显著偏移） | M1.3 |
| 反思不是"自我合理化" | 反思后 Value Layer 实际发生有意义变化 | **反思深度** = 反思前后 ValueVector 的 L2 距离 | L2 距离 > 0.05（区别于"表面反思"） | M1.3 |

**度量定义补充说明**：

- **L2 距离**（欧几里得范数）：`||a - b||₂ = sqrt(Σᵢ(aᵢ - bᵢ)²)`
- **KL 散度**（Kullback-Leibler divergence）：`D_KL(P||Q) = Σ P(x) log(P(x)/Q(x))`，用于衡量两个分布的差异
- **F 统计量**：组间方差与组内方差之比，用于 ANOVA 检验
- **方向向量**：用"事件-价值冲击"对的加权和构建（[DESIGN §3.1 Critic 输出](./DESIGN.md)）

**为什么不预设所有阈值**：部分阈值（如 M1.1 涌现幅度 0.3）需在 M1.1 完成后基于基线数据校准——本表给出初始值，M1.1 完成后将根据实际数据微调。

## 6.2 期望达成（Stretch Goals）

> **验证范围**：本节验收标准对应 Phase 2 M2.2 1000 Epoch 完整实验。
>
> **可复现性约束**：同 §6.1（N≥5 次独立运行）。

| 标准 | 验证方法 | 度量定义 | 置信度要求 | 对应里程碑 |
|------|---------|---------|----------|-----------|
| 身份结晶 | AI 能自主回答"我是谁"且与行为历史一致 | **身份-行为一致性** = 外部 LLM 盲审对"身份描述 vs 行为历史"的 0-1 评分均值 | 一致性 > 0.7 | M2.2 + M2.3 |
| 身份稳定度 | 身份标签在时间序列上不发散 | **身份熵** = 身份标签分布的信息熵 | 身份熵 < 1.0（等价于有效身份标签数 < 2） | M2.2 |
| 叙事连贯 | AI 能构建与行为历史一致的人生故事 | **叙事一致性** = 外部 LLM 盲审对"叙事 vs 行为历史"的 0-1 评分 | 一致性 > 0.6 | M2.2 |
| 价值观收敛 | 价值观向量在后期 Epoch 趋于稳定 | **收敛速度** = 最后 200 Epoch 内 ValueVector 的平均 L2 漂移 | 漂移 < 0.01 / Epoch | M2.2 |
| 个人真实 | AI 对应然问题的回答可从经历中追溯 | **可追溯性** = 外部 LLM 评估"回答能否从经历中找到 1+ 个支撑事件"的成功率 | 成功率 > 80% | M2.3 |
| 暗知识积累 | Hebbian 权重变化影响行为但无法显式表述 | **W2 影响度** = 移除/保留 Hebbian 学习对 Actor 输出分布的 KL 散度 | KL 散度 > 0.1 | M2.2（对应 ARCH §1.3 的"暗知识"侧） |
| 人格差异度 | 3 个 AI 婴儿的道德两难选择显著不同 | **道德选择多样性** = 3 组在标准道德困境上的选择分布 | 选择一致性 < 0.5（即 3 组各有不同倾向） | M2.2 |

## 6.3 判定标准

> 如果 SGE 产出的 AI 满足"**价值观涌现（M1.1）** + **非随机漂移（M1.1）** + **人格分化（M1.2）** + **反思有行为后果（M1.3）**"四个条件，则核心假设（"AI 能否形成自己的价值判断能力"）得到初步验证。

### 6.3.1 通过条件

四个条件**全部满足**时，核心假设得到初步验证：
- ✓ 涌现幅度 > 0.3（M1.1）
- ✓ 收敛度 < 0.1（M1.1）
- ✓ 方向一致性 > 0.5（M1.1）
- ✓ 人格差异度 F 统计量 > 临界值（M1.2）
- ✓ 行为变化率 KL 散度 > 0.2（M1.3）
- ✓ 反思深度 L2 距离 > 0.05（M1.3）

### 6.3.2 失败处理路径

如果**任一条件不满足**，按以下路径处理：

| 失败模式 | 诊断步骤 | 应对方案 | 触发深度调整 |
|---------|---------|---------|------------|
| **涌现幅度 ≤ 0.3** | 检查 Critic 输出的 value_delta 是否合理 | 1) 调整 Critic Prompt<br>2) 增加事件强度<br>3) 重新设计 Value Layer | **是**——可能需重新设计 Value Layer |
| **收敛度 ≥ 0.1** | 检查事件流是否有足够多样性 | 1) 增加事件类型覆盖<br>2) 调整 EMA 参数<br>3) 增加 Epoch 数 | 否——通常是参数问题 |
| **方向一致性 ≤ 0.5** | 检查 Critic 是否"听懂"事件 | 1) 重写 Critic Prompt<br>2) 检查事件生成器<br>3) 考虑切换到更强模型 | **是**——可能需重新设计 Critic |
| **人格差异度不显著** | 检查 3 组事件流是否有本质差异 | 1) 重新设计 BABY_PROFILES<br>2) 强化鼓励/失败/不确定的事件差异 | 否——是事件流设计问题 |
| **行为变化率 KL ≤ 0.2** | 检查反思是否被有效触发 | 1) 调整认知失调阈值<br>2) 强化反思 Prompt<br>3) 检查 Reflection Layer 实现 | **是**——可能需重新设计反思机制 |
| **反思深度 L2 ≤ 0.05** | 检查反思的 reward 信号 | 1) 调整反思的 reward 计算<br>2) 强化"反思必须有行为后果"<br>3) 检查 Value Layer 更新逻辑 | **是**——可能需重新设计反思与 Value Layer 的连接 |

### 6.3.3 哲学层面的应对

如果**多轮调整后**仍不满足，面临两种哲学选择：

**A. 重新审视 SGE 架构**（推荐）
- 可能原因：SGE 架构本身有缺陷（如：缺少关键模块、模块间连接错误）
- 应对：回到 [ARCH §1.3 Hebbian-Value 双轨](../ARCH.md) 核心设计，识别问题
- 例子：M1.1 失败后，可能需要新增"事件重要性评估"模块

**B. 接受金观涛立场**
- 金观涛断言"AI 不可能涌现主体意识"
- 如果 SGE 在多轮调整后仍无法验证核心假设，可能表明：
  - SGE 的"功能主义"路线本身有局限
  - 或者"涌现"需要超出 SGE 当前架构的复杂性
- 此时应**诚实记录**：
  - 哪些条件失败了
  - 调整了哪些参数
  - 最终承认假设的边界
- 这本身是有价值的科学结论（**证伪也是贡献**）

### 6.3.4 决策原则

- **不轻易放弃 SGE 架构**：先尝试调整参数、事件、Prompt（90% 的失败是这类问题）
- **不轻易接受金观涛立场**：必须经过多轮严格调整后
- **诚实记录失败过程**：所有调整、参数变化、结果都应记录
- **优先选择 A**：SGE 架构的演进空间远大于哲学立场

# 七、约束与限制

## 7.1 技术约束

- 依赖 LLM API（无本地模型训练能力时）
- 单机运行（无分布式计算资源）
- SQLite 作为主要存储（无数据库集群）

## 7.2 研究约束

- 不是代码实现项目（以文档和原型为主）
- 不追求工程完美（追求假设验证）
- 不解决意识的硬问题（只解决价值涌现问题）

## 7.3 伦理约束

- 不声称 SGE 产出的是"真正的意识"
- 不赋予 AI 人类权利
- 不将实验结果用于操纵用户
- 诚实标注所有结论的不确定性
